Best
Type E
풀 와이드 탭 (Full-width tab)

👍 장점

  • LDS 가이드와 톤이 잘 맞아 전체 화면과의 일관성이 높음
  • 2개 탭 구조에서 가장 단순하게 인지되어 현재 화면 목적에 적합
  • 상단 한 줄에서 상태 확인과 전환이 가능해 시선 이동이 짧음
  • 굵기 + 인디케이터로 선택 상태가 표현되어 현재 위치 파악이 빠름
  • 장식 요소가 적어 화면 밀도가 낮고 콘텐츠 집중도가 높음

👎 단점

  • 비선택 탭의 대비가 약하면 클릭 가능한 요소로 즉시 인지되지 않을 수 있음
  • 버튼형 대비 터치 가능 영역이 명확히 드러나지 않을 수 있음
  • 미니멀한 표현이라 일부 사용자에게는 명확성이 부족하게 느껴질 수 있음
Type E-2
스크롤 탭 (Scroll tab)

👍 장점

  • Type E와 동일하게 LDS 톤과의 일관성 유지
  • 탭이 균등 분할되지 않아 탭 수 증가 시에도 유연하게 대응 가능
  • 구조가 단순해 시각적 피로도가 낮음

👎 단점

  • 현재 2개 탭 기준에서는 Type E 대비 구조적 이점이 크지 않음
  • 균등 분할이 아니기 때문에 경우에 따라 균형감이 덜 안정적으로 느껴질 수 있음
  • 터치 영역 인지 문제는 Type E와 동일하게 존재
추천
Type A
탭 버튼 스타일

👍 장점

  • 버튼형 세그먼트 패턴으로 대부분의 사용자에게 익숙해 초기 학습 없이 사용 가능
  • 선택/비선택 상태가 명확하게 구분되어 현재 위치 인지가 쉬움
  • 터치 영역이 넓어 조작 안정성이 높음
  • 접근성 측면에서 비교적 안정적인 구조

👎 단점

  • 시각적으로 다소 무겁고, 최신 LINE 화면 톤과는 거리감이 있을 수 있음
  • 컴포넌트 자체의 존재감이 강해 상단 영역이 답답하게 느껴질 수 있음
  • 콘텐츠보다 UI 요소가 강조되는 구조
추천
Type B
라디오버튼 스타일(기존 디자인)

👍 장점

  • 선택/비선택 상태가 명확하게 표현되어 오인 가능성이 낮음
  • 라디오 버튼 형태로 선택 결과를 확실하게 전달

👎 단점

  • 탭 전환 UI보다 설정/폼 선택 UI로 인지될 가능성이 있음
  • 상단 네비게이션 패턴과는 성격이 달라 화면 목적과 부조화 발생 가능
  • 시각적으로 다소 분절되어 탐색 흐름이 끊길 수 있음
Type C
왼쪽 타이틀 + 소형 탭

👍 장점

  • 상단 공간을 거의 차지하지 않아 콘텐츠 영역을 넓게 확보 가능
  • 보조 조작 요소처럼 보여 화면이 비교적 덜 복잡하게 느껴질 수 있음

👎 단점

  • 탭 UI로 인지되기 어려워 발견 가능성이 낮음
  • 터치 영역이 작아 조작성이 떨어질 수 있음 (최소 44px 권장 대비 부족 가능성)
  • 선택 상태 표현이 약해 현재 위치 인지가 어려움
  • 주요 전환 UI로 쓰기에는 정보 전달력이 부족
Type D
아이콘 + 텍스트 탭

👍 장점

  • 아이콘을 활용해 텍스트 이해를 보조할 수 있음
  • 시각적 요소가 추가되어 구조를 직관적으로 전달할 가능성 있음

👎 단점

  • 텍스트만으로 충분한 상황에서 아이콘이 중복 정보가 될 수 있음
  • 아이콘 해석이 필요한 경우 인지 부담이 증가할 수 있음
  • 시각 요소가 많아지면서 상단 UI가 무거워질 수 있음
  • 아이콘 의미가 명확하지 않으면 오히려 이해를 방해할 수 있음
